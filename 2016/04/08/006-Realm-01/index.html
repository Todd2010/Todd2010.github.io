Realm – SQLite和Core Data的替代品

  • 如果你准备使用Swift语言来开发应用，不管是全新的Swift应用还是Objective-C与Swift混合的应用，那么你应该考虑使用Swift版本的Realm。
  • Realm不支持Objective-C版本与Swift版本在同一个项目中共存。

Reaml能以高效、安全、快速的方式来编写App的数据模型层。

先大致浏览一下Realm数据模型的简单定义与使用：

如果您的App正在使用Core Data并且打算换成Realm，可以参考这篇文章 Migrating an App from Core Data to Realm。

准备工作

  • 运行App的基本要求：iOS >= 7.0, OS X >= 10.9, WatchKit
  • Xcode版本号 >= 6.4
  • 支持Objective-C, Swift 1.2 & Swift 2.x

Xcode中导入Realm框架

1、安装最新版本的 CocoaPods （ >= 0.39.0 ）
2、新建 Podfile 文件，编辑 Podfile 文件并输入 

platform :ios, '8.0'

target 'RealmExtension' do
pod 'Realm', '~> 0.98.3'
end

3、在命令行中执行 pod install
4、关掉RealmExtension.xcodeproj工程，用RealmExtension.xcworkspace文件重新打开工程。

Realm Browser

我们还提供了一个名叫 Realm Browser 的独立Mac App来读取并且编辑 .realm 数据库。

有两种方式来安装 Realm Browser 应用：

  • Realm Browser Github 地址 下载代码，直接运行就能生成 Realm Browser 应用
  • Realm Browser Mac App Store 地址 通过Mac App Store安装。

运行 Realm Browser 应用，点击 Tool -> Generate demo database 可以生成一个有样本数据的测试数据库。

对于如何查看App在iOS设备中的 .realm 文件，可以参考StackOverflow上的这个解决办法 How to find a Realm file if your App is on iOS Device。

Xcode Realm插件（安装意义不大）

Realm提供了一个Xcode插件(Github 地址)，主要有两个功能：

  • 提供了Realm Model的模板。（替代方案：在新建Model的时记得继承 RLMObject）
  • Debug的时候能够在 Variables View 中能够看到 RLMObject 、 RLMResult 以及 RLMArray 对象的具体内容。（替代方案：NSLog可以在控制台中打印具体内容）
