Realm – SQLite和Core Data的替代品

  • 如果你准备使用Swift语言来开发应用，不管是全新的Swift应用还是Objective-C与Swift混合的应用，那么你应该考虑使用Swift版本的Realm。
  • Realm不支持Objective-C版本与Swift版本在同一个项目中共存。

Reaml能以高效、安全、快速的方式来编写App的数据模型层。

先大致浏览一下Realm数据模型的简单定义与使用：

如果您的App正在使用Core Data并且打算换成Realm，可以参考这篇文章 Migrating an App from Core Data to Realm。

开始

准备工作

  • 运行App的基本要求：iOS >= 7.0, OS X >= 10.9, WatchKit
  • Xcode版本号 >= 6.4
  • 支持Objective-C, Swift 1.2 & Swift 2.x

Xcode中导入Realm框架

1、安装最新版本的 CocoaPods （ >= 0.39.0 ）
2、新建 Podfile 文件，编辑 Podfile 文件并输入 

platform :ios, '8.0'

target 'RealmExtension' do
pod 'Realm', '~> 0.98.3'
end

3、在命令行中执行 pod install
4、关掉RealmExtension.xcodeproj工程，用RealmExtension.xcworkspace文件重新打开工程。

Realm Browser

我们还提供了一个名叫 Realm Browser 的独立Mac App来读取并且编辑 .realm 数据库。

有两种方式来安装 Realm Browser 应用：

  • Realm Browser Github 地址 下载代码，直接运行就能生成 Realm Browser 应用
  • Realm Browser Mac App Store 地址 通过Mac App Store安装。

运行 Realm Browser 应用，点击 Tool -> Generate demo database 可以生成一个有样本数据的测试数据库。

对于如何查看App在iOS设备中的 .realm 文件，可以参考StackOverflow上的这个解决办法 How to find a Realm file if your App is on iOS Device。

Xcode Realm插件（安装意义不大）

Realm提供了一个Xcode插件(Github 地址)，主要有两个功能：

  • 提供了Realm Model的模板。（替代方案：在新建Model的时记得继承 RLMObject）
  • Debug的时候能够在 Variables View 中能够看到 RLMObject 、 RLMResult 以及 RLMArray 对象的具体内容。（替代方案：NSLog可以在控制台中打印具体内容）

数据模型（Models）

使用常规Objective-C的类(Class)和属性(Property)就可以来定义Realm的数据模型。通过简单的继承 RLMObject 或者继承一个已经存在的 RLMObject 模型类来创建一个新的 RLMObject 数据模型。Realm模型对象在功能上几乎与其他常规Objective-C对象一样 —— 您可以为Realm的模型类添加方法(Method)和协议(Protocol)。主要的限制是一个Realm数据模型对象只能在其被创建的那个线程中使用（同一个Realm数据模型对象无法跨线程共享）；另外，您无法直接重写任何存储属性的 Setter 和 Getter 。

可以用 RLMArray 来建立对象之间的关系（如一对多，多对多），还可以添加目标类型的属性来实现嵌套数据结构(nested data structure)。 

// 狗狗的数据模型中通过添加Person类型的属性来实现嵌套数据结构
#import <Realm/Realm.h>

@class Person;

// 狗狗的数据模型
@interface Dog : RLMObject
@property NSString *name;
@property Person   *owner;
@end


// 狗狗主人的数据模型中通过定义RLM_ARRAY_TYPE(Dog)来建议人与狗一对多的关系
#import <Realm/Realm.h>

@class Dog;

RLM_ARRAY_TYPE(Dog) // 定义RLMArray<Dog>

// 狗狗主人的数据模型
@interface Person : RLMObject
@property NSString      *name;
@property NSDate        *birthdate;
@property RLMArray<Dog> *dogs;
@end

支持的数据类型

Realm支持以下的属性类型：BOOL、 bool、 NSInteger、 int、 long、 long long、 float、 double、 NSString、 NSDate （精度为秒）、NSData 以及被特殊类型标记的NSNumber 。

CGFloat由于在不同的平台上精度不同，所以Realm不支持此类型。（在32位系统上为float，在64位系统上为double）

您可以使用 RLMArray<Object *><Object> 和 RLMObject 的子类来建立诸如一对多、多对多的关系模型。
在Xcode 7以及之后的版本中，RLMArray支持编译阶段Objective-C 泛型(generics)的检测。下面解释一下 RLMArray<Object *><Object> 不同部分的意思和作用：

  • RLMArray：属性类型。（表示该属性是一个 RLMObject 类型的数组）
  • <Object *>：泛型特定化(generic specialization)。这可以在编译的时候检测出错误的对象类型。
  • <Object>：RLMArray遵守的协议。这可以让Realm在运行时确定数据模型的模式。 
#import <Realm/Realm.h>

@interface Dog : RLMObject
@property NSString *name;
@property NSInteger age;
@end


#import <Realm/Realm.h>

@class Dog;

RLM_ARRAY_TYPE(Dog)

@interface Person : RLMObject
@property NSString      *name;
// 如果dogs的类型为RLMArray<Dog>，那么在编译的时候，程序无法确定Dog类是否为RLMObject的子类，只有在运行时才能确定。
@property RLMArray<Dog> *dogs;

// 如果dogs的类型为RLMArray<Dog *><Dog>，那么在编译的时候，程序就能确定Dog类是否为RLMObject的子类，因此不能用@class Dog; 只能用#import "Dog.h"
@property RLMArray<Dog *><Dog> *dogs;
@end

关系

RLMArray 类似于 NSArray ，可以通过索引下标(indexed subscripting)来访问 RLMArray 中的对象。但和 NSArray 不同的是， RLMArray 中只能存放一个个 RLMObject 的子类对象。

  • 对一关系（To-One Relationships）
    不管是多对一还是一对一，只需要声明一个 RLMObject 子类属性即可。

  • 对多关系（To-Many Relationships）
    您可以通过定义一个 RLMArray 类型的属性来实现对多关系。
    如果要实现人与狗之间一对多的关系，首先需要定义一个 RLMArray<Dog> 类型，可以通过 RLM_ARRAY_TYPE 宏来创建一个协议，从而允许 RLMArray<Dog> 语法的使用。

允许给 RLMArray 的属性赋值 nil ，表示清空数组。

  • 反向关系（Inverse Relationships）
    链接不仅是单向的（unidirectional）还是相互独立的。就Person.dogs与Dog的链接和Dog.owner与Person的链接来说，独立性表现在为Person.dogs属性添加一个新的Dog实例并不会自动将Dog.owner的属性设置为该Person。
    Realm提供了一个API（ -linkingObjectsOfClass:forProperty: ）来建立反向链接。 
// Person和Dog是多对多的关系
// 在Dog类中，定义一个只读属性owners。（Realm不会存储只读属性）
@interface Dog : RLMObject
@property NSString *name;
@property NSInteger age;
@property (readonly) NSArray *owners;
@end

@implementation Dog
// owners与Person.dogs就建立了反向关系
// 访问owners就能获取某条狗的所有主人
- (NSArray *)owners {
    return [self linkingObjectsOfClass:@"Person" forProperty:@"dogs"];
}
@end

可选属性(Optional Properties)

可选属性是那些可以设置为 nil 的属性。比如 NSString 、NSData 以及 NSDate 类型的属性。如果不允许这些类型修饰的属性为空，那么您可以重写RLMObject子类的 +requiredProperties 方法。 

// 如果对name属性设置为nil将会抛出异常
// 对birthday属性设置nil是允许的
@interface Person : RLMObject
@property NSString *name;
@property NSDate *birthday;
@end

@implementation Person
+ (NSArray *)requiredProperties {
    return @[@"name"];
}
@end

如果要存储可空数字的属性，可以通过 NSNumber 来实现。由于Realm对不同类型的数字采取了不同的存储格式，因此设置可空的数字属性必须被标记为 RLMInt 、RLMFloat 、RLMDouble 或者 RLMBool 其中的一个类型，赋值给这些属性的值都会被转换为指定的类型。 

@interface Person : RLMObject
@property NSString *name;         // name属性不允许为空
@property NSNumber<RLMInt> *age;  // age属性允许设置为空（nil）
@end

@implementation Person
+ (NSArray *)requiredProperties {
    return @[@"name"];
}
@end

属性特性（Property Attributes）

在上面的代码中，您应该注意到Realm忽略掉了Objective-C的属性特性，如 nonatomic , atomic , strong , copy , weak 等，这么做是因为Realm有它自己的语义解释系统。因此，为了避免不必要的麻烦，我们建议您在Realm模型类中不要使用任何属性特性。

索引属性（Indexed Properties）

通过重写 +indexedProperties 方法来为某个数据模型中的属性建立索引。
索引属性能够以极小的插入代价来极大的提升匹配查询的速度（匹配查询采用的是 = 和 IN 操作符）。
索引属性的类型只能是 NSString , NSInteger , BOOL 和 NSDate。

@interface Book : RLMObject
@property float price;
@property NSString *title;
@end

@implementation Book
+ (NSArray *)indexedProperties {
  return @[@"title"];
}
@end

属性默认值（Default Property Values）

在每次RLMObject对象建立的时候，可以通过重写 +defaultPropertyValues 方法来为对象中的某些属性提供默认值。 

@interface Book : RLMObject
@property float price;
@property NSString *title;
@end

@implementation Book
+ (NSDictionary *)defaultPropertyValues {
    return @{@"price" : @0, @"title": @""};
}
@end

对象的自动更新

RLMObject实例会根据数据的改变来自动更新。比如，有N个对象都指向了某一条数据，若其中一个对象对该条数据有所修改，那么，这种修改也会直接反应到其他对象上去。

Dog *dog01 = [[Dog alloc] init];
dog01.name = @"小白";
dog01.age = 1;

[realm transactionWithBlock:^{
  [realm addObject:dog01];
}];

Dog *dog02 = [[Dog objectsWhere:@"age == 1"] firstObject];

//  dog01与dog02实际上是指向了同一条数据
NSLog(@"dog01 age: %ld", dog01.age);  // => 1
NSLog(@"dog02 age: %ld", dog02.age);  // => 1

[realm transactionWithBlock:^{
  dog02.age = 2;
}];

NSLog(@"dog01 age: %ld", dog01.age);  // => 2
NSLog(@"dog02 age: %ld", dog02.age);  // => 2

RLMObject的这个特性不仅让Realm保证了速度和效率，同时还让代码更加的简洁。比如，如果某段UI代码是基于RLMObject对象来实现的，那么在UI重绘之前，您就不需要重新刷新Realm对象。
如果您想要实现只要当Realm数据变动（增、删、改）就更新UI，那么可以查看下文中的 Realm 通知 和 KVO 。

主键（Primary Keys）

重写 +primaryKey 方法可以设置模型的主键。有了主键之后，不仅能保证对象的唯一性，还能提高查询和更新数据的速度。对象的主键一旦赋值，无法修改。 

@interface Person : RLMObject
@property NSInteger id;
@property NSString *name;
@end

@implementation Person
+ (NSString *)primaryKey {
    return @"id";
}
@end

忽略属性（Ignored Properties）

重写 +ignoredProperties 方法可以让Realm不存储数据模型的某些属性。Realm不会干涉这些属性的常规操作，可以重写它们的 setter 和 getter 方法。 

@interface Person : RLMObject
@property NSInteger tmpID;
@property NSString *firstName;
@property NSString *lastName;
@property (readonly) NSString *name; // 只读属性将被自动忽略（相当于忽略属性）
@end

// name和tmpID都是忽略属性
@implementation Person
+ (NSArray *)ignoredProperties {
    return @[@"tmpID"];
}
- (NSString *)name {
    return [NSString stringWithFormat:@"%@ %@", self.firstName, self.lastName];
}
@end

模型继承（Model Inheritance）

Realm允许模型继承。

写操作（Writes）

对象的所有更改（添加、修改和删除）都必须通过写入事务(transaction)完成。

Realm对象在未存储到数据库之前，您可以把它当作常规的Objective-C对象使用。如果想要在线程间共享这些对象或者在应用重启后还能使用它们，那么您就应该将它们存储到Realm数据库中 – 这个操作必须在写入事务中完成。

由于写入事务会产生不可忽略的性能消耗，因此您应当尽量避免大量写入事务的操作。

由于写入事务可能会有失败的情况（比如磁盘满了），Realm建议在写入事务中通过加入NSError来实现错误处理。- transactionWithBlock: , - transactionWithBlock:error:。 一般情况下，使用- transactionWithBlock: 即可。

创建对象

创建一个新的对象有多种方式。 

// 模型
@interface Dog : RLMObject
@property NSString *name;
@property NSInteger age;
@end


// 第一种方式：与创建Objective-C类相似
Dog *myOneDog = [[Dog alloc] init];
myOneDog.name = @"Rex";
myOneDog.age = 10;

// 第二种方式：利用initWithValue方法。（字典）
Dog *mySecondDog = [[Dog alloc] initWithValue:@{@"name" : @"Pluto", @"age" : @3}];

// 第三种方式：利用initWithValue方法。（数组，属性次序要相同）
Dog *myThirdDog = [[Dog alloc] initWithValue:@[@"Pluto", @3]];

需要注意的是：所有的必须属性（非可选属性，非忽略属性）都要在对象写入Realm数据库之前赋值。

嵌套对象（Nested Objects）

如果RLMObject模型对象中含有 RLMObject或者 RLMArray 类型的属性，那么可以通过嵌套的字典或者数组来递归的进行设置。 

// 已经初始化了三个Dog对象：myOneDog，mySecondDog，myThirdDog
// 第一种方式：数组中嵌套Dog对象
Person *aPerson = [[Person alloc] initWithValue:@[@"Jane", @30, @[myOneDog, mySecondDog, myThirdDog]]];

// 第二种方式：字典中嵌套Dog对象
Person *aPerson = [[Person alloc] initWithValue:@[@"Jane", @30, @[@[@"myForthDog", @4],
                                                                 @[@"myFifthDog", @5]]]];

添加数据（Adding Objects）

向Realm数据库中添加数据的步骤如下： 

// 创建对象
Person *author = [[Person alloc] init];
author.name    = @"大卫·福斯特·华莱士";

// 获取默认的 Realm 实例(每个线程只需要使用一次即可)
RLMRealm *realm = [RLMRealm defaultRealm];

// 通过事务将数据添加到 Realm 中
[realm beginWriteTransaction];
[realm addObject:author];
[realm commitWriteTransaction];

当对象被存储到Realm数据库之后，您在写入事务中对该对象做的任何修改都会被保存起来。当写入事务执行完毕之后，在使用相同Realm实例的其他线程中的操作才能够依次被执行。

请注意，如果进程中存在多个写入操作，那么单个写入操作将会阻塞其余的写入操作，并且还会锁定该操作所在的当前线程。因此，建议您最好把写入操作放到独立的线程中去执行（非主线程、UI线程）。

由于Realm采用了 MVCC 设计架构，因此，读取操作并不会因为写入事务正在进行而受到影响。
Realm建议您应当采用批量化的写入事务，而不是采用多次少量的写入事务。

更新对象（Updating Objects）

Realm提供了几种更新对象的方式，您可以根据具体的使用场景来选择最合适的更新对象的方式。

  • 一般方式（在写入事务中通过设置对象的某个属性来完成更新操作） 
RLMRealm *realm = [RLMRealm defaultRealm];

Dog *dog = [[Dog alloc] initWithValue:@{@"name" : @"dogNameBefore", @"age" : @(1)}];
[realm transactionWithBlock:^{
    [realm addObject:dog];
}];
NSLog(@"Dog before: %@", dog);    // Dog before: Dog { name = dogNameBefore; age = 1; }

[realm transactionWithBlock:^{
    dog.name = @"dogNameAfter";
}];
NSLog(@"Dog after: %@", dog);    // Dog after: Dog { name = dogNameAfter; age = 1; }

RLMResults *allDogs = [Dog allObjects];
NSLog(@"All dogs: %@", allDogs);    // [0] Dog { name = dogNameAfter; age = 1; }
  • 通过主键来创建或更新对象的方式
    如果您的数据模型中设置了主键，您可以通过 - [RLMRealm addOrUpdateObject:] 方法或者 - [RLMObject createOrUpdateInRealm:withValue:] 方法来更新对象，假如当对象不存在的时候，Realm会把该对象插入到数据库中。 
// 创建一个带有主键的对象（id是Book模型的主键）
Book *book01 = [[Book alloc] init];
book01.title = @"Book 01 Title";
book01.price = @9000;
book01.id = @1;

// 通过 id==1 更新该书籍
// 如果 id==1 的书籍已经存在数据库中了，那么会更新book01对象的title和price内容
// 如果 id==1 的书籍在数据库中不存在，那么会把book01对象作为一个新的数据插入到数据库中
[realm beginWriteTransaction];
[realm addOrUpdateObject:book01];
[realm commitWriteTransaction];

// [realm beginWriteTransaction];
// [Book createOrUpdateInRealm:realm withValue:@{@"id": @1, @"price": @9000.0f}];
// [realm commitWriteTransaction];

如果您的模型中没有设置主键，那么在调用以上两种方法时都会抛出异常，导致App崩溃。
nil对于可选属性来说是一个有效的值，也就是说，如果您把可选属性的值更新为nil，那么对象的该属性值就是为空。

  • 键值编码方式
    RLMObject 、RLMResult 以及 RLMArray 都遵守键值编码(Key-Value Coding)机制。这种方式对于在运行时修改属性是非常适合的。
    将KVC运用在批量修改集合中对象的属性值是一种极佳的方式，这样就不用再去遍历集合访问一个个的对象属性。 
RLMResults *persons = [Person allObjects];
[[RLMRealm defaultRealm] transactionWithBlock:^{
    // 将第一个人的isFirst属性值设置为YES
    [[persons firstObject] setValue:@YES forKeyPath:@"isFirst"];
    // 将每个人的planet属性值都设置为Earth
    [persons setValue:@"Earth" forKeyPath:@"planet"];
}];

删除对象（Deleting Objects）

在写入事务中调用 -[RLMRealm deleteObject:] 方法就可以删除某个具体的对象。
在写入事务中调用 -[RLMRealm deleteObjects:] 方法就可以删除一组对象。
在写入事务中调用 -[RLMRealm deleteAllObjects] 方法就可以删除Realm中的所有内容。（当数据库中的所有数据都被删除之后，Realm文件占用的空间仍会保留以便日后高效的复用）

Book *cheeseBook = ... // 存储在Realm中的某个对象
// 在写入事务中删除cheeseBook
[realm beginWriteTransaction];
[realm deleteObject:cheeseBook];
[realm commitWriteTransaction];

RLMResults *allBooks = [Book allObjects];
// 从Realm中删除所有Book对象
[realm beginWriteTransaction];
[realm deleteObjects:allBooks];
[realm commitWriteTransaction];

// 从Realm中删除所有数据
[realm beginWriteTransaction];
[realm deleteAllObjects];
[realm commitWriteTransaction];

查询

查询操作能够返回只包含 RLMObject 这一种类型的对象数组 RLMResults 。

查询操作的结果不是数据的拷贝，如果在写入事务中修改了查询结果里某个对象的属性值会直接修改数据库中的数据。

Realm的所有查询操作和属性访问操作采用的都是懒加载方式。比如说，查询语句的执行会被延迟到查询结果的使用。

一旦执行语句被执行之后，或者 通知模块（notification） 被添加之后，在后台线程的帮助下，RLMResults 会时刻保持更新。

从Realm中检索对象最基本和常用的方法是 + [RLMObject allObjects]，这个方法会返回所有RLMObject类型的对象。

条件查询

如果你熟悉 NSPredicate 的话，那么其实您已经知道Realm的查询方法了。您可以通过断言字符串、断言格式化字符串（predicate format string）或者 NSPredicate 实例来完成查询操作。 

// 使用断言字符串查询
RLMResults<Dog *> *tanDogs = [Dog objectsWhere:@"color = '棕黄色' AND name BEGINSWITH '大'"];

// 使用 NSPredicate 查询
NSPredicate *pred = [NSPredicate predicateWithFormat:@"color = %@ AND name BEGINSWITH %@",
                                                     @"tan", @"B"];
tanDogs = [Dog objectsWithPredicate:pred];

Realm不仅有自己的一套断言格式（NSPredicate Cheatsheet），还支持苹果的 NSPredicate 的查询语句（Predicate Programming Guide）。
